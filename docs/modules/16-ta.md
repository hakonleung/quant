# 16 — TA (Technical Analysis, beta)

> Pure price/volume analysis. Lives next to `analyze` (sentiment) but is
> **its own top-level surface** — the terminal command and the BFF route
> family use the `ta` namespace, not `analyze ta`.

## 1. Surfaces

| Surface  | Endpoint / command                        | Purpose                                          |
| -------- | ----------------------------------------- | ------------------------------------------------ |
| Terminal | `ta`, `ta <code>`, `ta sector <id\|name>` | Single-stock TA + sector fan-out                 |
| Web BFF  | `POST /api/ta/analyze_one`                | Single-stock TA (LLM, paid)                      |
| Web BFF  | `GET  /api/ta/analyze_one?code=`          | Cached single-stock TA (cache miss → 404)        |
| Web BFF  | `POST /api/ta/analyze_many`               | Sector fan-out + LLM summary                     |
| Action   | `analyze.ta`                              | Single-stock; cacheKey `analyze.ta:<code>`       |
| Action   | `analyze.ta.many`                         | Sector; cacheKey `analyze.ta.many:<sorted-codes>` |

## 2. Sector fan-out

`POST /api/ta/analyze_many` is **NestJS-side fan-out**:

1. For each `code` in body, call Flight `analyze_ta_one` in parallel.
   Each per-stock result is already cached (Python side), so a sector
   whose members were analysed recently is fast.
2. Reduce to a `TaSectorAnalysis`:
   - `trendBreakdown`: `{up,down,sideways}` member counts
   - `overallDirection`: argmax over the breakdown
   - `overallConfidence`: average confidence of members in the
     overall direction
   - `members[]`: slim `TaSectorMember` per stock (drops level lists
     and patterns; keeps the leading R/S prices)
3. Build a sector narrative via `LlmService.completeJson` (scope `ta`).
   Prompt is `apps/api/src/modules/ta/prompts/sector-summary.prompt.ts`.
   On LLM failure, `summary` is empty — caller still gets the numerical
   aggregate, with caveats describing each member that errored.

The orchestration intentionally lives in NestJS rather than as a Python
RPC handler:

- per-stock TA already has its own Python cache + RPC (`analyze_ta_one`)
- Python does not hold an external LLM client (CLAUDE.md §2.1)
- Rule of Three: this is the first multi-stock TA caller; introduce a
  Python `analyze_ta_many` only when a third caller appears

## 3. Schemas

`packages/shared/src/types/ta.ts`:

- `TaAnalysisSchema` — single-stock view (unchanged).
- `TaSectorMemberSchema` — per-stock card embedded in the aggregate.
- `TaSectorAnalysisSchema` — aggregate response: `codes`,
  `trendBreakdown`, `overallDirection`, `overallConfidence`, `members[]`,
  `summary`, `caveats`, `cachedAt`.
