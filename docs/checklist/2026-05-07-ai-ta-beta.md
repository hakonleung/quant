# Review checklist — 2026-05-07 AI 量价 / 形态技术分析 (beta)

新增独立的 `analyze ta` 子指令，纯 90 日量价 + 图形 LLM 分析。优先 Kimi
Pro，缺 key 自动 fallback。包含 Python service / cache / RPC、NestJS
gateway、web BFF、TERM action + command + live runner 全链路。

## Provider 选择 + fallback (`quant_io/llm/`)

- [ ] `MOONSHOT_API_KEY` + `QWEN_API_KEY` 都在 → `build_llm_client(prefer_provider="moonshot").name == "moonshot"`
- [ ] 仅 `QWEN_API_KEY` → `build_llm_client(prefer_provider="moonshot").name == "qwen"`（自动 fallback，不再 raise）
- [ ] 全部 LLM key 缺失 → `LLM_FAILED`，错误信息列出 `tried` 的 env 名
- [ ] `prefer_provider="bogus"` → `LLM_FAILED`（catalog 中无该 provider）
- [ ] `build_llm_client_chain(prefer_provider="moonshot")` 在三 key 都有时返回 `["moonshot", "qwen", "deepseek"]` 顺序
- [ ] `FallbackLlmClient.complete_json`：A 抛 LLM_FAILED → 自动调用 B → 成功；日志中可见一条 `ta_llm_fallback`
- [ ] `FallbackLlmClient.complete_json`：A 抛 `INVALID_ARGUMENT` → **立即上抛**，不会调用 B
- [ ] `FallbackLlmClient` 全链失败 → 抛出错误的 `details["attempts"]` 含每段 provider+code+error

## Python TA service (`quant_core/services/ta_service.py`)

- [ ] `analyze_one(code)` 首次调用：`kline_service.get_last_n(code, 90)` 被调用，prompt 中含 `date,open,high,low,close,volume,ma5,ma10,ma20,ma60` CSV 头
- [ ] 二次调用同 `code`：cache 命中，**不**再调 LLM
- [ ] `bypass_cache=True`：强行重新调 LLM
- [ ] 不存在的 code → `STOCK_NOT_FOUND`
- [ ] code 有元数据但仓库无 K 线 → `KLINE_DATA_MISSING`
- [ ] LLM 返回非法 JSON → `LLM_FAILED`
- [ ] LLM 输出包 ```` ```json ... ``` ```` markdown fence → 正确剥离后解析
- [ ] LLM 输出 `trend.direction = "moonbeam"` → `LLM_FAILED`
- [ ] LLM 输出 `trend.horizon_days = 0` → `LLM_FAILED`
- [ ] LLM 输出 `trend.confidence = 1.7` → 自动 clamp 到 `1.0`
- [ ] LLM 输出 `support_levels[0].strength = "uncertain"` → 该条被丢弃，其他条目保留
- [ ] 写入的 `TaAnalysis.provider` 对应当前生效的 provider 名（`moonshot` / `qwen` / 链组合 `moonshot+qwen+deepseek`）

## Parquet TA cache (`quant_cache/parquet_ta_cache.py`)

- [ ] 首次 `put` 创建 `data/ta/<code>.parquet`，schema = `TA_SCHEMA`
- [ ] 重复 `put` 同 `(code, asof)` → 行覆盖（不重复，文件原子替换）
- [ ] `expires_at = asof + 2 days @ 00:00 UTC`：clock 到达后 `get` 返回 `None`
- [ ] schema_version 不一致的旧行 → `get` 返回 `None`（不抛错）
- [ ] `invalidate(code)` → 文件删除；`get` 返回 `None`
